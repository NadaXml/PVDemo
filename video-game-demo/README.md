# 互动影游 - 使用说明

## 🎬 项目简介

这是一个基于 H5 的分支剧情互动影游引擎，支持：
- 视频节点播放
- 分支选择（带倒计时）
- 多结局系统
- 自定义故事配置
- 节点路径追踪

## 📁 文件结构

```
video-game-demo/
├── index.html          # 游戏页面
├── game.js             # 游戏引擎核心代码
├── server.js           # HTTP 服务器
├── story-config.json   # 故事配置示例
└── README.md           # 本文件
```

## 🚀 快速开始

### 1. 启动服务器

```bash
node server.js
```

然后访问 `http://localhost:80` 或 `http://你的IP:80`

### 2. 使用示例故事

项目内置了一个完整的示例故事，包含：
- 1 个起始节点
- 7 个剧情节点
- 10 个不同结局

点击「开始游戏」即可体验！

## 📝 创建自己的故事

### 节点类型

#### 1. 普通节点（带选择）
```json
{
  "id": "节点唯一标识",
  "name": "节点显示名称",
  "video": "视频文件路径或URL",
  "duration": 10,  // 播放多少秒后显示选择
  "question": "选择问题文本",
  "choices": [
    { "text": "选项A", "next": "下一个节点ID", "key": "A" },
    { "text": "选项B", "next": "另一个节点ID", "key": "B" }
  ],
  "timeout": 10,  // 选择倒计时（秒）
  "timeoutChoice": "超时时默认选择的next节点"
}
```

#### 2. 结局节点
```json
{
  "id": "ending_1",
  "name": "结局名称",
  "video": "结局视频路径",
  "isEnding": true,
  "endingTitle": "结局标题",
  "endingDesc": "结局描述文本"
}
```

### 完整配置示例

参考 `story-config.json` 文件，它包含一个完整的故事结构示例。

### 上传自定义配置

1. 按照 `story-config.json` 的格式创建你的故事
2. 点击页面上的「📁 上传视频配置」按钮
3. 选择你的 JSON 配置文件
4. 点击「开始游戏」

## 🎮 游戏操作

| 操作 | 说明 |
|------|------|
| 鼠标点击 | 选择分支选项 |
| 键盘 A/B/C... | 快捷键选择对应选项 |
| 跳过剧情 | 直接显示当前节点选择 |
| 重置 | 重新开始游戏 |

## 📹 视频准备

### 视频格式
- 推荐格式：MP4 (H.264编码)
- 推荐分辨率：1920x1080 或 1280x720
- 推荐时长：每个节点 5-15 秒

### 视频存放
- 在线视频：直接使用 URL（如 `https://example.com/video.mp4`）
- 本地视频：放在项目目录下，使用相对路径（如 `./videos/scene1.mp4`）

## 🎨 自定义样式

编辑 `index.html` 中的 `<style>` 部分可以修改：
- 颜色主题
- 按钮样式
- 字体大小
- 动画效果

## 🔧 高级功能

### 倒计时自动选择
每个选择可以设置倒计时，超时后自动选择默认选项：
```json
"timeout": 10,
"timeoutChoice": "默认下一个节点ID"
```

### 节点路径追踪
页面底部显示玩家走过的节点路径，方便追踪剧情走向。

### 结局统计
达成结局时显示：
- 经历节点数
- 做出选择数
- 游戏时长

## 💡 创作建议

1. **故事结构**：建议先画流程图，确定所有节点和分支
2. **视频拍摄**：每个节点对应一段视频，注意转场连贯性
3. **选择设计**：每个选择应该有明显不同的后果导向
4. **结局数量**：建议至少 3-5 个不同结局增加重玩价值
5. **测试路径**：确保每个分支都能到达结局，没有死胡同

## 🐛 常见问题

### 视频无法播放
- 检查视频格式是否为 MP4 (H.264)
- 检查视频路径是否正确
- 检查服务器是否正确运行

### 配置加载失败
- 确保 JSON 格式正确（可用在线 JSON 验证工具检查）
- 确保 `startNode` 指向的节点存在
- 确保所有 `next` 指向的节点都已定义

### 跨域问题
如果视频放在其他域名，确保服务器设置了 CORS 头：
```
Access-Control-Allow-Origin: *
```
